# Design Review Checklist

## Token and Style System

- Colors use semantic/source/member tokens where available
- Spacing, radius, shadows, and typography are tokenized
- Inline style literals are minimized

## Visual Hierarchy

- Primary action is obvious
- Headings and content density are scannable
- Supporting information does not overpower core task

## States and Interaction

- Hover, focus, active, disabled states exist
- Loading, empty, and error states are defined
- Keyboard focus is visible and logical

## Responsiveness

- Content does not overflow at mobile widths
- Tap targets are usable on touch screens
- Text remains readable without clipping

## Accessibility

- Color contrast appears sufficient for text and controls
- Inputs/buttons have labels
- Regions and dynamic updates use appropriate ARIA roles
